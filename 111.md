# 基于ROS的四旋翼无人机轨迹规划与避障系统

## 摘要

本文介绍了一个基于ROS的四旋翼无人机轨迹规划与避障系统。该系统采用分层架构设计，前端使用A*算法进行路径搜索，后端使用Minimum Snap算法进行轨迹优化。系统支持静态和动态障碍物环境下的实时轨迹规划与重新规划，通过状态机管理轨迹生成和执行过程，保证轨迹的安全性和平滑性。

## 1. 地图和障碍物生成

### 1.1 地图参数设置

系统采用三维栅格地图表示环境，地图参数包括：
- 地图尺寸：30m × 30m × 4m（长×宽×高）
- 地图分辨率：0.2m
- 地图坐标系：world坐标系
- 地图原点：(-15, -15, 0)m

### 1.2 静态障碍物生成

静态障碍物采用随机生成策略，生成参数包括：
- 障碍物数量：30个
- 障碍物类型：长方体（BOX）
- 障碍物尺寸：宽度0.3-0.8m，高度3.0-7.0m
- 障碍物位置：随机分布在地图范围内，避开初始位置

静态障碍物生成算法：
1. 在地图范围内随机选择障碍物中心位置
2. 检查障碍物是否与初始位置冲突（距离小于1.0m）
3. 检查障碍物是否与其他障碍物冲突（距离小于1.0m）
4. 生成障碍物点云数据，用于路径规划
5. 生成障碍物Marker数据，用于可视化

### 1.3 动态障碍物生成

动态障碍物采用圆形轨迹运动策略，生成参数包括：
- 动态障碍物数量：3个
- 障碍物类型：长方体（BOX）
- 障碍物尺寸：宽度0.3-0.8m，高度3.0-7.0m
- 运动轨迹：圆形轨迹
- 运动速度：1.0-2.0 m/s
- 圆形半径：3.0-5.0m
- 更新频率：60Hz

动态障碍物生成算法：
1. 定义三个圆形轨迹中心点，避开静态障碍物区域
2. 为每个动态障碍物分配一个圆形轨迹
3. 随机设置圆形半径和运动速度
4. 计算角速度：ω = v / r
5. 设置初始位置和初始角度
6. 生成障碍物点云数据和Marker数据

动态障碍物运动模型：
x(t) = x_center + r × cos(ωt + φ) y(t) = y_center + r × sin(ωt + φ) z(t) = height / 2

其中：
- (x_center, y_center)为圆形轨迹中心
- r为圆形半径
- ω为角速度
- φ为初始相位
- height为障碍物高度

### 1.4 障碍物可视化

系统使用RViz进行障碍物可视化，发布两种类型的Marker：
- 静态障碍物Marker：蓝色，半透明，发布到/random_complex/static_obstacle_markers
- 动态障碍物Marker：红色，半透明，发布到/random_complex/obstacle_markers

Marker更新频率：60Hz，确保动态障碍物运动平滑显示。

## 2. 无人机运动控制

### 2.1 状态机管理

系统采用状态机管理轨迹生成和执行过程，定义了五种状态：

1. **INIT（初始化状态）**
   - 等待接收里程计数据
   - 接收到里程计数据后切换到WAIT_TARGET状态

2. **WAIT_TARGET（等待目标状态）**
   - 等待接收目标点
   - 接收到目标点后切换到GEN_NEW_TRAJ状态

3. **GEN_NEW_TRAJ（生成新轨迹状态）**
   - 执行轨迹生成算法
   - 生成成功切换到EXEC_TRAJ状态
   - 生成失败保持GEN_NEW_TRAJ状态

4. **EXEC_TRAJ（执行轨迹状态）**
   - 执行生成的轨迹
   - 检查是否需要重新规划
   - 满足条件切换到REPLAN_TRAJ状态

5. **REPLAN_TRAJ（重新规划状态）**
   - 从当前轨迹点开始重新规划
   - 生成成功切换到EXEC_TRAJ状态
   - 生成失败切换到GEN_NEW_TRAJ状态

### 2.2 重新规划策略

系统支持在线重新规划，重新规划条件包括：

1. **轨迹完成**
   - 当前时间超过轨迹总时间
   - 切换到WAIT_TARGET状态

2. **接近目标点**
   - 当前位置与目标点距离小于2.0m
   - 不需要重新规划

3. **偏离轨迹**
   - 当前位置与轨迹起始点距离大于1.5m
   - 触发重新规划

4. **时间阈值**
   - 轨迹执行时间超过1.0s
   - 触发重新规划

### 2.3 里程计数据接收

系统订阅/visual_slam/odom话题，接收无人机状态信息：
- 位置信息：x, y, z坐标
- 速度信息：vx, vy, vz速度
- 姿态信息：四元数表示的姿态

里程计更新频率：100Hz

## 3. 前端后端轨迹规划

### 3.1 整体架构

系统采用前端+后端的双层轨迹规划架构：

**前端：A*路径搜索算法**
- 功能：在栅格地图中搜索从起点到终点的路径
- 输出：离散的路径点序列
- 特点：保证路径可行性，但路径不平滑

**后端：Minimum Snap轨迹优化算法**
- 功能：将离散路径点优化为平滑的连续轨迹
- 输出：多项式系数矩阵
- 特点：保证轨迹平滑性和连续性

### 3.2 轨迹生成流程

轨迹生成流程包括以下步骤：

1. **路径搜索**
   - 使用A*算法搜索从起点到终点的路径
   - 获得离散的路径点序列

2. **路径简化**
   - 使用RDP（Ramer-Douglas-Peucker）算法简化路径
   - 减少路径点数量，提高计算效率

3. **时间分配**
   - 为每个路径段分配时间
   - 考虑最大速度和最大加速度约束

4. **轨迹优化**
   - 使用Minimum Snap算法优化轨迹
   - 最小化加加速度（jerk）

5. **安全检查**
   - 检查轨迹是否与障碍物碰撞
   - 不安全则重新优化

6. **轨迹发布**
   - 发布多项式系数矩阵
   - 发布轨迹可视化Marker

## 4. 轨迹搜索

### 4.1 A*算法原理

A*算法是一种启发式搜索算法，结合了Dijkstra算法和贪婪最佳优先搜索的优点。算法通过评估函数f(n) = g(n) + h(n)来选择最优节点，其中：
- g(n)：从起点到节点n的实际代价
- h(n)：从节点n到终点的启发式估计代价
- f(n)：节点n的总估计代价

### 4.2 三维栅格地图

系统使用三维栅格地图表示环境：
- 地图尺寸：150 × 150 × 20（x × y × z）
- 网格分辨率：0.2m
- 障碍物标记：使用uint8_t数组，1表示障碍物，0表示自由空间

### 4.3 启发式函数

系统采用对角距离启发式函数：
h = d0 + d1 + d2 + (√3 - 3.0)×d0 + (√2 - 2.0)×d1

其中：
- d0, d1, d2为排序后的距离分量
- 使用对角距离提高搜索效率

### 4.4 邻居节点扩展

系统采用26-邻域扩展策略，每个节点有26个邻居节点：
- 6个面邻居：dx, dy, dz中只有一个非零
- 12个边邻居：dx, dy, dz中有两个非零
- 8个角邻居：dx, dy, dz都非零

邻居节点代价：
- 面邻居：1.0
- 边邻居：√2 ≈ 1.414
- 角邻居：√3 ≈ 1.732

### 4.5 算法流程

A*算法流程：

1. **初始化**
   - 将起点加入开集
   - 设置起点的gScore为0
   - 计算起点的fScore

2. **主循环**
   - 从开集中弹出fScore最小的节点
   - 将节点加入闭集
   - 扩展邻居节点
   - 更新邻居节点的gScore和fScore
   - 将邻居节点加入开集

3. **终止条件**
   - 找到终点：回溯路径
   - 开集为空：搜索失败

4. **路径回溯**
   - 从终点开始，沿着cameFrom指针回溯到起点
   - 得到从起点到终点的路径

### 4.6 算法优化

系统采用以下优化策略：
1. **双向搜索**：从起点和终点同时搜索，提高搜索效率
2. **tie-breaker**：打破平局，减少搜索节点数量
3. **早期终止**：找到终点后立即终止搜索

## 5. 路径简化

### 5.1 RDP算法原理

RDP（Ramer-Douglas-Peucker）算法是一种路径简化算法，通过递归地划分线段，保留关键点，删除冗余点。

算法原理：
1. 连接起点和终点，形成一条线段
2. 找到距离线段最远的点
3. 如果最远距离大于阈值，则在该点处分割路径
4. 递归处理两个子路径
5. 如果最远距离小于阈值，则删除中间所有点

### 5.2 距离计算

点到线段的距离计算：
d = ||(P - A) × (P - B)|| / ||A - B||

其中：
- P为待判断的点
- A为线段起点
- B为线段终点
- ×表示叉积
- ||·||表示范数

### 5.3 简化效果

路径简化效果：
- 减少路径点数量：从数千个点减少到几十个点
- 提高计算效率：减少轨迹优化计算量
- 保持路径形状：保留关键特征点

### 5.4 简化参数

路径简化参数：
- 简化阈值：0.05m
- 最小路径点数：2个（起点和终点）

## 6. 安全检查

### 6.1 安全检查原理

安全检查用于验证生成的轨迹是否与障碍物碰撞。系统通过采样轨迹上的点，检查每个点是否在障碍物内。

### 6.2 采样策略

系统采用均匀采样策略：
- 采样间隔：0.01s
- 采样点数：每个轨迹段采样约100个点
- 采样总点数：所有轨迹段采样点数之和

### 6.3 碰撞检测

碰撞检测算法：
1. 计算采样点的三维坐标
2. 将三维坐标转换为栅格索引
3. 检查栅格是否被障碍物占据
4. 如果被占据，则返回不安全的轨迹段

### 6.4 重新优化策略

如果检测到不安全的轨迹段，系统采用重新优化策略：

1. **插入中间点**
   - 在不安全段的中间插入一个中间点
   - 中间点坐标为：(start + end) / 2
   - 将中间点"圆整"到最近的栅格中心

2. **重新优化**
   - 使用新的路径点重新分配时间
   - 重新执行Minimum Snap优化
   - 重新进行安全检查

3. **迭代优化**
   - 最多迭代10次
   - 如果10次后仍不安全，则放弃该轨迹段

### 6.5 安全保证

系统通过以下机制保证轨迹安全：
1. **密集采样**：0.01s采样间隔确保不遗漏碰撞
2. **迭代优化**：最多10次迭代确保找到安全轨迹
3. **栅格化障碍物**：障碍物膨胀一个栅格，确保安全余量

## 7. 无人机模型参数和约束

### 7.1 无人机模型参数

系统采用四旋翼无人机模型，主要参数包括：
- 质量：0.98kg
- 最大速度：3.0m/s
- 最大加速度：2.0m/s²
- 最大加加速度：未明确限制，通过轨迹优化控制

### 7.2 轨迹约束

轨迹优化约束包括：

1. **位置约束**
   - 起点位置：p(0) = p₀
   - 终点位置：p(T) = p_T
   - 中间点位置：p(t_i) = p_i

2. **速度约束**
   - 起点速度：v(0) = v₀
   - 终点速度：v(T) = v_T
   - 最大速度：|v(t)| ≤ 3.0m/s

3. **加速度约束**
   - 起点加速度：a(0) = a₀
   - 终点加速度：a(T) = a_T
   - 最大加速度：|a(t)| ≤ 2.0m/s²

4. **连续性约束**
   - 位置连续：p_i(T_i) = p_{i+1}(0)
   - 速度连续：v_i(T_i) = v_{i+1}(0)
   - 加速度连续：a_i(T_i) = a_{i+1}(0)

### 7.3 多项式表示

系统使用分段多项式表示轨迹：
p(t) = Σ c_i × t^i

对于Minimum Snap，使用7阶多项式：
p(t) = c₀ + c₁t + c₂t² + c₃t³ + c₄t⁴ + c₅t⁵ + c₆t⁶ + c₇t⁷

多项式系数数量：8个系数 × 3个维度 = 24个系数/轨迹段

### 7.4 时间分配

系统采用基于速度和加速度的时间分配策略：

1. **短距离**
   - 距离 ≤ d_scope = 2.0×v_max / a_max = 3.0m
   - 时间：t = √(d / a_max)
   - 运动模式：加速-减速

2. **长距离**
   - 距离 > d_scope
   - 时间：t = t_scope + (d - d_scope) / v_max
   - 运动模式：加速-匀速-减速

其中：
- v_max为最大速度（3.0m/s）
- a_max为最大加速度（2.0m/s²）
- t_scope为加速时间（3.0s）
- d_scope为加速距离（3.0m）

### 7.5 轨迹优化目标

系统采用Minimum Snap优化目标：
min ∫₀^T ||jerk(t)||² dt

其中：
- jerk(t) = d³p(t)/dt³为加加速度
- ||·||表示欧几里得范数
- T为轨迹总时间

优化目标的意义：
- 最小化加加速度，提高轨迹平滑性
- 减少无人机机械磨损
- 提高飞行舒适性

### 7.6 二次规划求解

系统将轨迹优化问题转化为二次规划问题：
min ½xᵀHx + fᵀx s.t. Ax ≤ b

其中：
- x为多项式系数向量
- H为Hessian矩阵
- f为梯度向量（通常为零向量）
- A为约束矩阵
- b为约束边界

系统使用OSQP求解器求解二次规划问题。
